# aTimeLogger 同步工具 V3

第三版继承第二版功能，支持 Google Calendar → aTimeLogger，以及 Toggl ↔ aTimeLogger 双向同步。

## Windows

双击：

```text
run_atimelogger_sync_choose_date.pyw
```

## Linux

在本文件夹中运行：

```bash
python3 run_atimelogger_sync_choose_date.py
```

## 命令行

同步今天：

```bash
npm run sync
```

测试模式：

```bash
npm run sync:dry-run
```

图形界面的 `Toggl 双向同步` 页面支持反复点击 `测试模式`。测试不会写入或覆盖记录。

`自动同步` 页面可以分别选择：

```text
Google Calendar → aTimeLogger
Toggl → aTimeLogger
aTimeLogger → Toggl
```

登录后同步、打开脚本时同步、固定时间同步和立即后台同步都会按已保存的选择运行。

Windows 创建计划任务被拒绝时，会自动改用当前用户本地定时器，无需管理员权限。

完整说明请阅读：

```text
ATIMELOGGER_SYNC_GUIDE.zh-CN.md
TOGGL_ATIMELOGGER_V3_GUIDE.zh-CN.md
ATIMELOGGER_NOTION_WORKER_GUIDE.zh-CN.md
```

本文件夹中的 `.dev.vars` 包含当前本地同步配置，可能包含账号密码和私密 iCal 地址，请安全保管。
